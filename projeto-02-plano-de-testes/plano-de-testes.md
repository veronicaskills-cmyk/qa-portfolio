# Plano de Testes – Aplicação Web

## 1. Introdução
Este plano de testes descreve a estratégia, o escopo e os critérios utilizados para a execução dos testes na aplicação web selecionada, com o objetivo de garantir a qualidade, funcionalidade e confiabilidade do sistema.

---

## 2. Objetivo
Garantir que as principais funcionalidades da aplicação funcionem conforme o esperado, identificando falhas antes da entrega ao usuário final.

---

## 3. Escopo dos Testes

### 3.1 Funcionalidades que serão testadas
- Login de usuário
- Cadastro de usuário
- Navegação entre páginas
- Validação de campos obrigatórios
- Mensagens de erro e sucesso

### 3.2 Funcionalidades fora do escopo
- Testes de performance
- Testes de segurança avançados
- Testes de compatibilidade mobile

---

## 4. Tipos de Teste
- Teste funcional
- Teste de regressão
- Teste exploratório
- Teste de usabilidade (básico)

---

## 5. Ambiente de Teste
- Sistema Operacional: Windows 10
- Navegador: Google Chrome
- Tipo de aplicação: Web

---

## 6. Critérios de Entrada
- Aplicação disponível para teste
- Requisitos definidos
- Ambiente configurado

---

## 7. Critérios de Saída
- Todos os casos de teste executados
- Bugs críticos corrigidos ou documentados
- Relatório final de testes elaborado

---

## 8. Papéis e Responsabilidades
- Analista de QA: Planejamento, execução e documentação dos testes
- Desenvolvedor: Correção de defeitos identificados

---

## 9. Riscos
- Mudanças de requisitos durante os testes
- Instabilidade do ambiente
- Prazo reduzido para execução

---

## 10. Aprovação
Este plano de testes deve ser aprovado pela equipe responsável antes do início da execução dos testes.
